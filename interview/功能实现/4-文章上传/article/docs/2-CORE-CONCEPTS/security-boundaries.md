# 安全边界

- 当前无登录、授权和多用户隔离，任何能访问服务的人都可以创建、修改和删除文章。
- API 入口执行输入校验、同源校验、限流和安全响应头。
- 正文服务端使用白名单清洗，不能只信任浏览器输出。
- OSS 对象先以 `private` 上传，服务端完成大小、MIME、文件魔数和 HEAD 校验后才切换为 `public-read`。
- AccessKey、STS Token 和其他密钥禁止使用 `NEXT_PUBLIC_` 前缀，也不能提交到仓库。
- fake OSS 只用于测试，不能被误认为生产存储。

公网部署前必须补充认证、授权、持久化限流、密钥托管、审计和恢复能力。
